decade

这题在我一开始看到的时候就感觉似曾相识，就是第一个正则匹配式，最开始是在code-breaking挑战赛中遇到过——无参数RCE，也就是构造a(b(c()))模式的函数达到RCE，但是在这题里禁止了下划线和数字，还在第二个正则里限制掉了很多函数。

可以写一个fuzz脚本来看看过滤掉了哪些函数，get、local、time、rand等好构造的函数都被禁用掉了，就连几个读文件的函数都被禁用了，但好在还有一个file函数，用于将文件内容输入数组中，我们可以再用pos() next()之类的函数来操作数组的下标来得到我们想要的内容。

好在scandir()这个函数还是在的，但是却发现getcwd()用不了了，那我们要怎么才能获得当前的路径，当然一个点.就是当前路径，但我们却不能输入点，但我们可以用chr(46)的方式得到一个点，但是我们又不能输入数字46，那怎么得到一个46呢？来看一个payload

为什么这里会输出一个46呢？其实并不是百分之百得到46的，crypt首先对phpversion()得到的数字进行了一次加密，加密后的字符串有可能在末尾出现一个点，然后再用strrev把它换到第一位，用ord换成ascii码，就可以得到一个点了。有了一个点，那么我们就可以通过scandir(.)得到两个点切换到上一层目录！因为scandir在linux系统下返回的数组前两个是”.”和”..”，那用一个next(scandir(chr(ord(strrev(crypt(phpversion()))))))就可以得到两个点，再使用chdir切换到上一级目录。

因为在我本机里flag.php文件在scandir中位于第三位而不是最后一位，所以用end不能直接获取到，我就加了一个array_reverse来处理了一下。（实际题目里应该是位于最后一个的）

eazysql

这题过滤了逗号和or两个关键的东西，or被过滤意味着无法通过访问information_schema.tables来得到表名，但在高版本的mysql中可以通过mysql.innodb_table_stats来得到表名，逗号被过滤可以使用join来绕过。

先记录一下join的用法

0’ union select * from (select 1)a join (select 2)b join (select 3)c #

等同于 0’ union select 1,2,3 #

再记录一下在高版本中的mysql库innodb_table_stats表的用法

但是这题做了限制，无法读取到列名，所以还得再做一次无列名注入。

无列名注入的用法是

select from a.3(select 1,2,3 union select from “表名”)a;
这样就可以读取出表中第三列的所有数据

而这题的正则又过滤了union select 但union%0bselect可以绕过
所以最后可构造payload

0’ union%0bselect (select 1)z join (select i.3 from(select from (select 1)a join (select 2)b join (select 3)c join union%0b select from fl111aa44a99g)i limit 1 offset 1)x join (select 3)v join (select 4)n %23
