360CTF复赛 流量分析

这个流量分析题目是在之前没有遇到过的做法，所以我并没有找到获取flag的途径，虽然知道解密流量得到传输的数据，但却离正确步骤越来越远。

首先打开流量包选择导出对象查看http流量，发现大多数格式都是一样的，打开前几个流量包看看
在第一个流量包中发现了php代码

通过百度这串代码，发现是冰蝎的马，再看了看后面的步骤，就是传递一个aes密钥给服务端，然后把客户端和服务端传送的流量都用这个aes密钥加密，而密钥在流量包中也可以直接看到

所以可以找几个流量中的数据解密一下看看具体进行了什么操作，我解密了大多数流量发现都是列当前目录下文件的，没有什么作用，然后就找到一个SQLite format 3,发现是一个sqlite数据库文件

百度搜索了一下这种文件的打开方式，是使用sqlite database browser

打开后可以看到flag字段，但value是看不到的，要解密才能看到。

后来看了wp才了解了这是存储谷歌浏览器账号密码的一个数据库文件，可以通过离线的方式导出密码。需要使用用户登录密码解密master key file 获得master key。再用master key解密DPAPI blob（也就是数据库文件中无法直接查看到的数据内容）。

所以我们首先要获得DPAPI blob，也就是从数据库文件中提取出我们要的数据的字段内容，可以用下面的这个脚本。

然后就是要找到master key file，刚好这题就是在最后一个流量中有这样的一个路径

响应包中的这一串数据就是master key file的内容，把它复制成一个文件（不要前面的1d4和后面的0）所以现在我们还差最后一步，就是得到解密master key file所需要的登录密码，这是在我比赛时就找到了但不知道有啥用的password.txt的内容。。。。

第322个流量中的相应包内容提取出来经过aes解密再base64解密得到password

最后这些解密所需要的条件我们都已经有了，要用一个工具windows password recovery来解密

选择Utils -> DPAPI Decoder and Analyser -> Decrypt DPAPI data blob

设置DPAPI blob file为python脚本提取出来的文件

master key file设置为刚刚从流量包中复制出来的文件

SID就是之前路径中的S-1-5-21-2127750816-4215895028-2373289296-1001
password就是p@ssw0rd
