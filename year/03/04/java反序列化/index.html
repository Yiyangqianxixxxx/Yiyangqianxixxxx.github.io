从一道ctf题学习java反序列化

Java反序列化漏洞和php反序列化漏洞有着差不多的意思，php反序列化时会执行对应的魔术方法__wakeup()，而Java反序列化时会执行readObject()方法，所以如果readObject()方法被恶意构造的话，就有可能导致命令执行。

Java的序列化是通过 ObjectOutputStream 类的writeObject()方法

Java的反序列化是通过 ObjectInputStream 类的 readObject()方法

实现java.io.Serializable接口才可被反序列化，且所有属性必须是可序列化的（用transient关键字修饰的属性除外，不参与反序列化过程）

一个简单的反序列化Demo

User.java

package serialize;

import java.io.Serializable;

public class User implements Serializable{
    private String name;

    public void setName(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
    private void readObject(java.io.ObjectInputStream stream) throws Exception{
        stream.defaultReadObject();
        //调用默认的readObject方法
        Runtime.getRuntime().exec("calc.exe");
    }
}

Main.java

package serialize;

import java.io.*;

public class Main{
    public static void main(String[] args) throws Exception{
        User user = new User();
        user.setName("fortheone");

        FileOutputStream fout = new FileOutputStream("user.bin");
        // 打开user.bin作为文件
        ObjectOutputStream out = new ObjectOutputStream(fout);
        // 打开一个文件输入流
        out.writeObject(user);
        // 文件输入序列化数据
        out.close();
        FileInputStream fin = new FileInputStream("user.bin");
        ObjectInputStream in = new ObjectInputStream(fin);
        in.readObject();
        in.close();
        fin.close();
    }

}

上面的代码展示了将一个对象序列化后写入文件中，然后反序列化读取出来，因为反序列化操作，执行了User的readObject方法，调用了Runtime.getRuntime().exec()来弹出计算器。
