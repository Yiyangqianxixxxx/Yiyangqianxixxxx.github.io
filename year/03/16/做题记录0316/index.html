这道题目还是蛮有意思的，至少丰富了自己的一部分还不太懂的知识点。

今天扫描器不知道怎么抽风了，题目的git泄露都扫不到，很怪，被迫换字典。

开局是一个留言板，选择发帖要登录

这里提示账号是 zhangwei 密码是 zhangwei*** 爆破最后三位得到密码 zhangwei666

看到留言什么的首先想到的就是xss，xss确实有的，不过没啥用，这个是留言给自己看的，没有bot访问，然后就是扫目录，找到git泄露，下载下来。

下载下来的是个不完整的源码，可以用git回滚到历史版本。

git log –reflog

看到历史版本有三个，然后用命令

git reset –hard

这里选择最上面的那个

git reset –hard e5b2a2443c2b6d395d06960123142bc91123148c

这里两个操作一个是创建帖子分类，一个是在帖子下面留言。

可以看到虽然第一步创建帖子的过程中对三个参数都进行了 addslashes 操作 单引号会被转义加上反斜杠\ 但是存入数据库的时候 会自动去掉这个反斜杠。

然后在第二步写留言的时候，直接从数据库中取出了 category 并且没有做任何处理，所以这里存在二次注入，只要在第一步的时候插入

category=1’,content=user(),/*

在第二步的时候留言

content=*/#

这样插入数据库中的留言就会变成

因为这里的sql语句是有换行符的，而#只能注释掉当前行的语句，所以需要使用/**/多行注释来注释掉语句，再用一个#注释掉content行的内容。

再根据wp一步步找文件读文件，读etc/passwd发现有个www用户，然后读操作历史，发现有个DS_Store文件，注意这里的文件内容太大，要用hex编码一下才能读出来。
