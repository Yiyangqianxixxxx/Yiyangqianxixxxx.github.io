3CTF内存取证题

周末巅峰极客和3CTF撞一起了，早上又还有实验课，上课时候看了看360的web题，一点思路都没有（到比赛结束也没想到）巅峰极客也没心思打，于是乎就琢磨起这道内存取证题。

内存取证题上次护网杯也做过了，但是没做出来，不过学习到了一些内存取证工具volatility的用法了。首先记录一些volatility的常用语法和内存取证做题的思路。

volatility 使用：
        volatility -f <文件名> -–profile=<配置文件> <插件> [插件参数] 
通过volatility --info获取工具所支持的profile，Address Spaces，Scanner Checks，Plugins

常用插件：
imageinfo：显示目标镜像的摘要信息，知道镜像的操作系统后，就可以在 –profile 中带上对应的操作系统
pslist：该插件列举出系统进程，但它不能检测到隐藏或者解链的进程，psscan可以
psscan：可以找到先前已终止(不活动)的进程以及被rootkit隐藏或解链的进程
pstree：以树的形式查看进程列表，和pslist一样，也无法检测隐藏或解链的进程
mendump：提取出指定进程，常用foremost 来分离里面的文件
filescan：扫描所有的文件列表
hashdump：查看当前操作系统中的 password hash，例如 Windows 的 SAM 文件内容
svcscan：扫描 Windows 的服务
connscan：查看网络连接
iehistory：查看ie浏览器浏览历史

所以第一步是要判断镜像的操作系统

命令: volatility -f <镜像文件名称> imageinfo

得到操作系统为windowsxpSP2x86

第二步我一般会查看当前镜像正在运行的进程

命令：volatility -f <镜像文件名> --profile=<操作系统> pslist

这里可以看到一个ie浏览器的进程，又联想到此题的题目描述

google语法真是太好了！！！

所以不难想到这题应该是与浏览器搜索有关，所以我们可以先把ie浏览器这个内存给dump下来查看。

volatility -f <镜像文件名> --profile=<操作系统> memdump -p <pid> -D ./

dump下来以后会在你选择的目录下生成一个(pid).dmp，所以我直接在这个文件里暴力搜索flag，但显然题目没有这么容易。

所以我选择了查看镜像中ie浏览器的浏览历史

volatility -f <镜像文件名> --profile=<操作系统> iehistory

可以看到搜索了github Syst2m，那我们也去github上找找看

这里有个360CTF的内容，但是好像又没有找到flag相关的东西

那其实题意就已经很明确了，再加上主办方提示了隐写，所以我们要找到一个图片，这个图片是F5隐写，密码是F5.txt中的内容。所以再去镜像里找图片。

命令：volatility -f <镜像文件名> --profile=<操作系统> filescan |grep "jpg\|png"

刚好看到有一个360.jpg，把它下载下来

命令：volatility -f <镜像文件名> --profile=<操作系统> dumpfiles -Q <物理偏移量> -D ./

但我们在使用F5隐写工具解密时却发现报错了，并且啥都没输出，我一度怀疑主办方给的文件是不是有问题，但是我仔细看了看报错内容：

file does not end with EOI

这个EOI的意思就是FFD9，也就是说该图片文件没有以FFD9结尾，但一般图片文件都是以FFD8开头FFD9结尾啊，难道说这个图片里还藏有其他的内容吗？所以我们利用hxd打开查看图片十六进制内容

居然发现图片末尾FFD9被填充了一堆空字符00…所以把它给去掉然后再去F5隐写工具解密
