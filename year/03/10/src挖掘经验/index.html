挖src一个月小总结一下

疫情推迟的开学让我终于在家坐不住了，想着试着挖挖src看看，但是苦于没有什么经验，所以只能去补天上找几个专属src练练手，运气还算不错，有一些收获。

老生常谈的信息搜集

说实话，信息搜集真的是我这段时间做的最多的一件事，远远多过我真正去测漏洞点的时间，因为主站怼不动，登录页面没东西，想搞点边缘业务点却时常是这样

搞着搞着，心态就崩了，不过看了很多师傅的经验总结后，我知道了就算是这些页面也是有可利用的点，所以整个渗透过程慢慢就变成了不断的fuzz。

所以还是从资产搜集开始，因为我挖的src不是什么大厂商，业务线不是特别长，业务也不是特别多，所以资产搜集的过程较简单。

备案查询

进入主站，拉到网页最底端，然后复制备案号，到备案信息查询网站查询其他的域名。

http://www.beian.miit.gov.cn/

SSL证书查询

直接把带有https的网站的域名复制过来搜索

https://crt.sh/

百度 谷歌 必应 github fofa各种搜索引擎

这里就不再介绍搜索的各种语法了，详见google hacking

记录一个学到的骚操作 shodan上根据图标来搜索资产

https://www.cesafe.com/5523.html

不过试用了一下，感觉搜到的资产还是和扫出来的差不多，可能是公司太小了？

目前用过最好的子域名搜集工具——oneforall

https://github.com/shmilylty/OneForAll

用法在文档上都有写，就不过多介绍了。

快速ip批量扫描

在测电信这种一大堆ip的时候，如果全部导入到nmap里来跑端口的话，不知道得跑多久，之前看到硬糖师傅分享的一个masscan+namp来扫描批量ip 的方法觉得很不错，并且经过实践后发现真的提速很多。先利用masscan扫描存活的ip端口，再导入nmap中识别端口的服务，因为nmap识别端口服务是特别快的。具体实现可以参考github上的代码。

让我暴躁的各种页面怎么办

在信息搜集过程中，最常碰到的就是nginx，各种403，404，welcome，这些都是因为url的根目录下没有设置页面，而在其url下有很多子目录放着很多服务接口什么的，这时候就需要爆破目录，类似于/admin/、/upload/、/config/等等 可能会扫出意想不到的惊喜

之前我用的是burpsuite来爆破js接口目录什么的，但是发现一个问题，不能自定义错误页面，不能递归爆破，虽然有着作战工具engagement tools，但是如果是要快速探测的话，实在不好用。

既然说到burp作战工具engagement tools 那就推荐一篇burp的用法

https://blog.gzsec.org/post/burpsuite-engagement-tools/

再介绍一下我最近用的比较多的目录爆破工具wfuzz，kali自带的，可自定义错误页面，递归扫描等等好用的功能，不过需要自备一份好字典，那就再把字典也分享一下吧。

https://github.com/TheKingOfDuck/fuzzDicts

碰到这种页面要咋办，利用搜索引擎，搜索该页面的域名，找到它的子目录下的功能，或者爆破。

碰到这种页面咋办，put漏洞，tomcat弱口令，找ajp端口。

碰到这种咋办，spring各种漏洞尝试，加env，加swagger-ui.html。

碰到这种咋办，weblogic各种漏洞扫一遍。

分享几个我挖到的案例

信息泄露员工邮箱

在子域名搜集的时候，碰到一个sentry页面，通过员工邮箱登录，尝试了几个搜集到的邮箱和弱口令没有成功，忘记密码的功能也是要向邮箱发送确认邮件，看似没有办法了。但是忽然想到去github上找信息泄露，于是在github上搜索该公司邮箱格式。

还真就被我搜到了，还是个弱口令，所以喜提低危100元，没有什么技术含量。

任意文件下载二连杀

有水印是当时挖的时候截图忘保存了，后面想复现发现环境没了。所以就去我的漏洞页面直接拿图了。开局扫到一个未授权访问的工程组件页面，里面有个上传功能和下载功能，上传绕不过jsp而且上传文件会到另一个访问不到的路径（不在webapp下，所以肯定无法解析jsp），尝试下载功能的时候抓包，发现路径

看到这个fileurl我就感觉有戏了，直接跳转到etc/passwd下发现可以直接读取

但是在我读etc/shadow的时候出现了问题，读不到，说明运行此程序的用户并不是root，那限制就有很多了，那么就读一下/home/user/.bash_history 爆破一下user，得到当前用户名，然后读到操作记录，在这个服务器上运行了多个版本的tomcat，依次去读web.xml server.xml之类的配置文件，数据库是搭在内网的服务器上的，无法直接连接。然后读到了几个用户名密码，在这个服务器的另一个端口找到登录页面。但是都无法登录。

然后发现了一个忘记密码页面，只要输入用户名，然后就会像绑定的手机号发送短信验证，尝试爆破的时候发现服务端只回复了一个 false，于是抓服务器响应包，修改为success放回去，居然就绕过去了！然后就任意密码重置了，进入页面后又发现一枚差不多的任意文件下载。

但是当我交漏洞的时候，居然重置密码页面被删了！我只能交个弱口令加任意文件下载了。。。

喜提高危加中危。（不要问我为什么没图，环境关了）

学了一天就挖到的未授权访问

前段时间感觉有点挖不动了，于是又偷学了波师傅们的经验，也就是这个spring框架下的swagger未授权访问。看到这个页面知道是spring框架，先测试下actuator服务，详情参见：http://www.hacksec.cn/Penetration-test/921.html

也就是在后面加个env，然后发现还是404，接着测swagger服务，也就是在后面加swagger-ui.html

then 测试这些接口是否可用，是否需要appkey之类的认证码，然后发现可任意使用，高危未授权，got it！

总结

其实在这个月挖洞的过程中，大多数都是没有什么技术含量的不断的信息搜集，然后fuzz，一个个漏洞测，但第一个月也算是小有收获了，继续努力。
